# Task Scheduler

Веб-сервер для управления списком задач (to-do list) на Go.

---

## Описание проекта

Приложение позволяет:

- создавать задачи;
- просматривать список задач;
- получать отдельную задачу по идентификатору;
- редактировать задачи;
- удалять задачи;
- отмечать задачи как выполненные;
- работать с повторяющимися задачами.

Для хранения данных используется SQLite.

Если через переменную окружения или .env-файл задан пароль, сервер включает JWT-аутентификацию.

---

## Реализованы:

- веб-сервер;
- инициализация базы данных и работа с SQLite;
- API для управления задачами;
- логика повторения задач:
  - `d <число>` — задача повторяется через указанное число дней. Максимально допустимое число равно 400. 
     Например:
      - `d 1` — каждый день;
      - `d 7` — каждую неделю;
      - `d 60` — каждые 60 дней.
  - `w <через запятую от 1 до 7>` — задача повторяется в указанные дни недели, где 1 — понедельник, 7 — воскресенье. 
     Например: 
      -  `w 7` — в ближайшее воскресенье;
      -  `w 1,4,5` — в ближайший понедельник, четверг или пятницу;
      -  `w 2,3` — в ближайший вторник или среду.
  - `m <через запятую от 1 до 31, -1, -2> [через запятую от 1 до 12]` —  задача повторяется в ближайшие указанные дни месяца. При этом вторая последовательность чисел опциональна и указывает на определённые месяцы. 
     Например:
      -  `m 4` — 4-е число каждого месяца;
      -  `m 1,15,25` — 1, 15 или 25-е число каждого месяца;
      -  `m -1` — последний день каждого месяца;
      -  `m -2` — предпоследний день каждого месяца;
      -  `m 3 1,3,6` —  3 января, марта или июня;
      -  `m 1,-1 2,8` — первый и последний дни февраля и авгуcта.
  - `y` — задача повторяется ежегодно. Этот параметр не требует дополнительных уточнений. При выполнении задачи дата перенесётся на год вперёд.
- поиск задач по тексту (регистрозависимый);
- поиск задач по дате;
- JWT-аутентификация;
- middleware для проверки токена;
- поддержка запуска через Docker.

## Переменные окружения

Проект поддерживает следующие переменные окружения:

| Переменная      | Описание                 | Значение по умолчанию |
| --------------- | ------------------------ | --------------------- |
| `TODO_PORT`     | порт запуска сервера     | `7540`                |
| `TODO_DBFILE`   | путь к файлу базы данных | `scheduler.db`        |
| `TODO_PASSWORD` | пароль для авторизации   | авторизация отключена |
| `JWT_SECRET`    | секретный ключ для JWT   | `super_very_secret`   |

## Локальный запуск

- Запуск сервера без авторизации:

```bash
go run ./cmd
```

- Запуск сервера с авторизацией:

Пример:

```bash
TODO_PASSWORD=12345 go run ./cmd
```

После запуска приложение будет доступно в браузере:
```text
http://localhost:7540
```
или на порту, указанном в `TODO_PORT`.

Если `TODO_PASSWORD` не пустой, произойдет перенаправление на:
```text
http://localhost:7540/login.html
```

## Запуск тестов

### 1. Запустить сервер

### 2. Получить токен (если используется авторизация):

Отправить POST-запрос:

```bash
curl -X POST http://localhost:7540/api/signin \
-H "Content-Type: application/json" \
-d '{"password":"12345"}'
```

Сервер вернёт:

```json
{
  "token": "your_jwt_token"
}
```

### 3. Указать токен в тестах (если используется авторизация):

В файле:

```text
tests/settings.go
```

установить:

```go
var Token = "your_jwt_token"
```

### 4. Указать параметры приложения в тестах (если не используются значения по-умолчанию):

В файле:

```text
tests/settings.go
```

установить:

```go
var Port = "{используемый порт}"
```

```go
var DBFile = "../{используемый путь}"
```

### 5. Запустить тесты

Обычный запуск:

```bash
go test -v ./tests
```

Запуск без использования кэша:

```bash
go test -count=1 -v ./tests
```

# Docker

Проект поддерживает запуск через Docker.

### Docker-образ содержит:

- скомпилированный веб-сервер;
- директорию `web` со статическими файлами фронтенда.

SQLite база данных хранится на хосте и подключается через volume, поэтому данные не теряются при перезапуске контейнера.

### Сборка Docker-образа:

```bash
docker build -t task-scheduler .
```

### Запуск контейнера:

Пример запуска:

```bash
docker run -p 7540:7540 \
-v "$(pwd)":/data \
-e TODO_PORT=7540 \
-e TODO_DBFILE=/data/scheduler.db \
task-scheduler
```

### Параметры:

- `-p 7540:7540` — проброс порта;
- `-v "$(pwd)":/data` — подключение текущей директории хоста для хранения SQLite базы;
- `TODO_PORT` — порт приложения;
- `TODO_DBFILE` — путь к базе данных внутри контейнера;
- `TODO_PASSWORD` — пароль для аутентификации.

При необходимости параметры запуска можно переопределить через переменные окружения.

После запуска приложение будет доступно в браузере:

```text
http://localhost:7540
```

## Структура проекта

- `cmd`
  - `main.go` — точка входа приложения
- `internal`
  - `api` — регистрация API маршрутов
  - `auth` — JWT аутентификация
  - `config` — инициализация конфигурационных данных
  - `db` — инициализация БД и реализация SQL запросов
  - `handlers` — HTTP обработчики
  - `helpers` — вспомогательные функции
  - `middleware` — middleware авторизации
  - `server` — конфигурация и запуск HTTP-сервера
  - `service` — бизнес-логика (повторения, даты)
- `tests` — автотесты проекта
- `web` — фронтенд (HTML/CSS/JS)

- `go.mod` - зависимости проекта
- `go.sum` - lock-файл зависимостей
- `Dockerfile` — сборка docker-образа
- `README.md` — документация по проекту
- `scheduler.db` - база данных SQLite
